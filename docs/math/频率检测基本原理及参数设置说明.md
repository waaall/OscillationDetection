# 频率检测基本原理及参数设置说明

## 1. 检测目标与基本原理

该功能用于判断信号中是否存在落在目标频带内的周期性振荡。

基本过程如下：

1. 按 `window_duration_s` 截取一段时间窗口。
2. 对窗口内数据做频谱分析（FFT），得到各频率成分的强度。
3. 在 `target_freq_range_hz` 内查找是否存在明显频率峰值。
4. 如果目标频带内的成分强度达到 `amp_threshold`，则判定该窗口检测到有效振荡。

例如，周期约 10 秒的振荡，其对应频率约为 `0.1 Hz`，频谱峰值通常会出现在 `0.1 Hz` 附近。

## 2. 主要参数说明

| 参数 | 含义 | 主要影响 | 设置要点 |
| --- | --- | --- | --- |
| `target_freq_range_hz = [min, max]` | 目标检测频带 | 决定算法重点关注哪些振荡 | `min` 越小，可检测越慢的振荡；`max` 越大，可检测越快的振荡 |
| `window_duration_s` | 单次分析窗口时长 | 决定低频能否被稳定识别，也影响频率分辨率 | 窗口越长，越容易看清慢振荡，但响应更慢 |
| `step_duration_s` | 相邻窗口滑动步长 | 决定结果刷新频率和时间定位精细度 | 步长越小，刷新越快、计算量越大 |
| `amp_threshold` | 幅值阈值 | 决定检测严格程度 | 阈值过低易误报，过高易漏报 |
| `sampling_rate_hz` | 采样率 | 决定可分析的最高频率 | 采样率越高，可检测的高频上限越高 |

### 2.1 `amp_threshold` 如何理解

`amp_threshold` 比较的是频谱主峰幅值，不是时域瞬时值。

- 对接近单频正弦的信号，可近似看作时域峰值幅度。
- 如果已知的是时域峰峰值，可先按其一半附近估计。
- 工程上通常把阈值设为期望检出主峰幅值的略小值，再结合样本回放微调。

## 3. 参数约束与设置原则

### 3.1 两个核心约束

| 约束项 | 条件 | 说明 |
| --- | --- | --- |
| 最高频率约束 | `target_freq_max_hz < sampling_rate_hz / 2` | 目标频带上限必须低于 Nyquist 频率，否则无法正确识别 |
| 最低频率约束 | `window_duration_s >= 3 / target_freq_min_hz` | 窗口至少应覆盖最低目标频率约 3 个周期，否则低频振荡难以稳定识别 |

可以用一句话记住：

- 高频能不能看见，主要取决于采样率。
- 低频能不能看清，主要取决于窗口长度。

### 3.2 推荐设置顺序

1. 先确定目标周期或目标频率范围，换算关系为 `频率(Hz) = 1 / 周期(秒)`。
2. 用目标最高频率检查采样率是否足够，并尽量留出一定余量。
3. 用目标最低频率反推窗口长度，工程上建议至少覆盖 3 个周期，更稳妥时可按 4 个周期设置。
4. 再根据刷新需求设置 `step_duration_s`，根据实际数据幅值调节 `amp_threshold`。

## 4. 输出字段说明

当前代码中的实际输出字段名为 `dominant_freq_hz`、`peak_amplitude`、`overall_peak_freq_hz`、`overall_peak_amplitude`。如果内部口头简写成 `domin_peak_*`，可按下表对应理解。

| 输出字段 | 含义 | 如何理解 |
| --- | --- | --- |
| `dominant_freq_hz` | 目标频带内主峰频率 | 目标频带内最强振荡对应的频率 |
| `peak_amplitude` | 目标频带内主峰幅值 | 与 `dominant_freq_hz` 对应的频谱峰值，用于和 `amp_threshold` 比较 |
| `overall_peak_freq_hz` | 全频段主峰频率 | 全频段最强振荡对应的频率 |
| `overall_peak_amplitude` | 全频段主峰幅值 | 与 `overall_peak_freq_hz` 对应的频谱峰值，用于判断是否为带外强振荡 |

### 4.1 频率精度说明

`dominant_freq_hz` 来自 FFT 频点，不是连续拟合值，因此会受窗口长度影响。

- 频率分辨率近似为 `delta_f ≈ 1 / window_duration_s`。
- 窗口越短，频率网格越粗，主峰频率越容易偏到相邻频点。
- 如果真实频率没有正好落在 FFT 频点上，主峰频率和主峰幅值都可能出现偏移。

如果目标频率比较明确，建议 `window_duration_s` 取目标周期的整数倍，工程上优先考虑约 4 个周期，同时仍满足 `window_duration_s >= 3 / target_freq_min_hz`。

## 5. 配置示例

假设参数如下：

| 参数 | 值 |
| --- | --- |
| `sampling_rate_hz` | `1.0` |
| `target_freq_range_hz` | `[0.012, 0.2]` |
| `window_duration_s` | `256` |
| `step_duration_s` | `1` |

这组配置表示：

- 每秒采样 1 次。
- 每次使用最近 256 秒的数据做一次频率分析。
- 每隔 1 秒滑动一次窗口并重新检测。
- 重点关注 `0.012 Hz` 到 `0.2 Hz` 之间的振荡。

约束检查如下：

- `0.2 < 1.0 / 2 = 0.5`，满足最高频率约束。
- `256 >= 3 / 0.012 = 250`，满足最低频率约束。

因此，这组配置是合理的，可以支持 `0.012 Hz` 到 `0.2 Hz` 范围内的振荡检测。

## 6. 常见误区

1. 只看单个参数是否合法，忽略参数之间的匹配关系。
2. 认为窗口越短越好。窗口过短虽然响应更快，但会看不清低频振荡。
3. 认为采样率够了就什么都能测。采样率只决定高频上限，低频识别仍依赖足够长的窗口。
4. 认为 `step_duration_s` 决定频率精度。实际上，频率分辨率更主要受 `window_duration_s` 影响。
