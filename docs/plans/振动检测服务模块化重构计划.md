# 低频工艺振荡检测改造设计文档（第一阶段：算法收敛与离线层重构）

## Summary
- 目标是将现有“50Hz 主频高精度跟踪”方案，收敛为“低频工艺信号振荡检测”方案，为后续 FastAPI 服务化做准备。
- 检测目标是电力/工艺信号中的低频振荡，主目标是异常报警，不再追求高精度频率估计。
- 第一阶段只完成算法边界、检测流程边界、离线 CSV 测试边界和统一输出格式设计；API 通信层只保留接口约束，不进入实现。
- 第一阶段删除 `ZeroCrossFreq`、`IpDFT`、`FrequencyRefinement`、相位输出和高精度相关测试，保留 FFT 作为唯一频域检测核心。

## Architecture
### 核心算法层
- 输入只接收等间隔采样数组 `samples` 和 `sampling_rate_hz`，不接触 CSV、时间戳、日志文件、图表、动画或 API。
- 固定处理流程为：`长度校验 -> 去均值 -> Hann 窗 -> rFFT -> 单边幅值归一化 -> 总峰值提取 -> 目标频带峰值提取`。
- 不再输出 `phase`、`rocof`、高精度修正结果。
- 标准输出只返回检测指标；离线层如需绘图，可通过调试开关额外获取 `spectrum_freqs` 和 `spectrum_amps`。

### 检测流程层
- 负责输入模式统一、时间戳解析、采样率校验、乱序处理、缺失值清理、可选重采样、窗口切分、状态映射。
- 同时支持两种输入模式：
  - `timestamp + value`
  - `value + sampling_rate_hz`
- 初始化时强校验 `window_duration_s >= 3 / target_freq_range_hz[0]`；不满足直接报配置错误，不进入窗口处理。
- 该层是未来离线 CSV 层和 FastAPI 层的共同下游。

### 外围离线 CSV 测试层
- 负责 JSON 配置、CSV 读取、滑窗回放、结果 CSV 输出、调试图表、测试信号生成。
- `step_duration_s` 只存在于该层，用于回放节拍；不进入核心算法层。
- 动画、绘图、示例信号生成都归入该层，不再混入检测流程层。

### API 通信协议层
- 本阶段不实现。
- 下一阶段只负责请求解析、环境变量配置注入、调用检测流程层和返回统一结果模型，不承载算法判断逻辑。

## Detection Spec
- 目标频带由 `target_freq_range_hz = [low, high]` 定义，要求 `0 < low < high < sampling_rate_hz / 2`。
- 核心算法同时计算两组峰值：
  - `overall_peak`：除 DC 外全有效频段最大谱峰。
  - `band_peak`：目标频带内最大谱峰。
- 报警判据固定为：`band_peak.amplitude >= amplitude_threshold`。
- 窗口必须覆盖目标最慢周期的 3 个周期；例如最低频率为 `0.0167Hz` 时，窗口时长至少约 `180s`。
- 频率精度不作为验收项；只考核“是否落入目标频带 + 幅值是否超阈”。

### Status 规则
- `alarm`：`band_peak` 存在，且 `band_peak.amplitude >= amplitude_threshold`
- `ok`：`band_peak` 存在，且 `band_peak.amplitude < amplitude_threshold`
- `out_of_band`：`band_peak` 未报警，但 `overall_peak.amplitude >= amplitude_threshold` 且 `overall_peak.freq_hz` 在目标频带外
- `insufficient_data`：窗口样本数不足，或输入总长度不足以形成第一个完整窗口
- `invalid_input`：频带非法、时间戳不可解析、采样率缺失、重采样失败、清理后数据为空等

### Reason 枚举
- `peak_above_threshold`
- `peak_below_threshold`
- `dominant_peak_outside_band`
- `window_too_short`
- `not_enough_samples`
- `invalid_frequency_range`
- `timestamp_parse_failed`
- `sampling_context_missing`
- `resample_failed`
- `empty_signal_after_cleanup`

## Interfaces And Data Contracts
### 核心算法输入
```json
{
  "samples": [0.12, 0.18, 0.21],
  "sampling_rate_hz": 1.0,
  "target_freq_range_hz": [0.0167, 0.2],
  "amplitude_threshold": 0.5,
  "include_spectrum": false
}
```

### 标准窗口输出
```json
{
  "status": "alarm",
  "reason": "peak_above_threshold",
  "window": {
    "window_id": 0,
    "start_index": 0,
    "end_index": 179,
    "start_time": "2026-03-14T10:00:00",
    "end_time": "2026-03-14T10:02:59",
    "sample_count": 180,
    "sampling_rate_hz": 1.0
  },
  "metrics": {
    "dominant_freq_hz": 0.0333,
    "dominant_period_s": 30.0,
    "peak_amplitude": 0.82,
    "overall_peak_freq_hz": 0.0333,
    "overall_peak_amplitude": 0.82,
    "threshold": 0.5
  },
  "meta": {
    "target_band_low_hz": 0.0167,
    "target_band_high_hz": 0.2,
    "source_mode": "csv"
  }
}
```

### 输出约束
- `window.start_time` 和 `window.end_time` 在无时间戳模式下为 `null`，但 `start_index`、`end_index`、`window_id` 必须始终存在。
- `metrics.dominant_freq_hz` 和 `metrics.peak_amplitude` 表示目标频带内峰值，不表示全频段峰值。
- `metrics.overall_peak_*` 作为诊断字段始终保留，用于解释 `out_of_band`。
- 不允许像当前实现那样跳过窗口；每个已生成窗口必须输出一条结果。
- 当输入总长度不足以形成首个完整窗口时，离线层仍输出一条 `insufficient_data` 结果，而不是空文件。

### 离线 CSV 输出列
```text
window_id,status,reason,start_index,end_index,start_time,end_time,sample_count,sampling_rate_hz,dominant_freq_hz,dominant_period_s,peak_amplitude,overall_peak_freq_hz,overall_peak_amplitude,threshold,target_band_low_hz,target_band_high_hz
```

### 离线 JSON 配置
```json
{
  "input": {
    "csv_path": "csv-data/input.csv",
    "time_column": "timestamp",
    "value_column": "signal",
    "timestamp_format": "%Y-%m-%d %H:%M:%S",
    "has_timestamp": true
  },
  "analysis": {
    "sampling_rate_hz": 1.0,

    "window_duration_s": 180,
    "amplitude_threshold": 0.5,
    "allow_resample": false
  },
  "replay": {
    "step_duration_s": 1.0
  },
  "output": {
    "result_csv_path": "csv-data/output.csv",
    "include_plot": true,
    "plot_dir": "plots/"
  }
}
```

### 时间戳与采样率规则
- 有时间戳且无 `sampling_rate_hz` 时，由检测流程层根据平均时间间隔推断采样率。
- 有时间戳且有 `sampling_rate_hz` 时，流程层做一致性校验；均值误差超过 `5%` 视为不一致。
- 时间间隔变异系数超过 `10%` 视为不规则采样。
- `allow_resample=false` 时，采样不一致或不规则直接返回 `invalid_input`。
- `allow_resample=true` 时，按目标采样率做线性插值重采样后再进入窗口计算。
- 数值缺失默认先做 `ffill` 再做 `bfill`；清理后仍为空则返回 `invalid_input`。
- 时间戳乱序默认先排序；若排序后存在重复且无法稳定重采样，则返回 `invalid_input`。

## Implementation Changes
- 将现有 FFT 模块收敛为纯计算器，只保留低频频带峰值检测相关逻辑。
- 将现有动画式 `OscillationDetection` 从主链路剥离，保留为离线可视化能力，不再承担检测流程职责。
- 将现有动态分析器收敛为离线 CSV 回放器，负责窗口推进和结果落盘，不再承载算法细节。
- 删除高精度频率修正链路及对应配置项、测试、文档描述。
- 删除过零检测链路及对应配置项、测试、文档描述。
- 保留 `SignalGenerator` 仅供离线测试和示例数据构造使用。

## Test Plan
- 单元测试：目标频带内峰值超过阈值时返回 `alarm`。
- 单元测试：目标频带内峰值低于阈值时返回 `ok`。
- 单元测试：全频段强峰在目标频带外时返回 `out_of_band`。
- 单元测试：窗口时长不足 3 个最慢周期时，初始化直接报配置错误。
- 单元测试：无时间戳但给定采样率的输入可正常处理。
- 单元测试：有时间戳且采样率一致的输入可正常处理。
- 单元测试：有时间戳但采样不规则且禁用重采样时返回 `invalid_input`。
- 单元测试：输入总长度不足一个完整窗口时输出一条 `insufficient_data` 结果。
- 集成测试：离线 CSV 回放输出行数等于窗口数，不丢窗口。
- 集成测试：离线绘图开启时只依赖核心算法返回的调试频谱，不重复实现 FFT。
- 回归测试：移除 `ZeroCrossFreq`、`IpDFT`、`FrequencyRefinement` 后，配置和 README 不再出现对应入口。

## Assumptions And Defaults
- V1 默认只做去均值，不做线性去趋势或高阶去趋势。
- V1 默认始终使用 Hann 窗，暂不开放为外部配置项。
- 频带配置只接受 Hz；周期只作为派生输出，不作为外部主配置。
- `out_of_band` 与 `alarm` 共用同一个幅值阈值，不新增第二套阈值。
- 单通道、单设备是固定前提，不引入多流状态管理。
- API 部分暂不实现，也不在本阶段定义网络错误码和鉴权方案。
