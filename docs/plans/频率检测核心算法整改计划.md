# 频率检测核心算法整改计划

## Summary
- 目标是修正 4 个已确认的问题：FFT 精化被窗函数污染、FFT 会被强谐波抢峰、ZC 在 `rising_only=False` 时频率翻倍、以及真实采样率与配置不一致时产生系统性频偏。
- 整改原则是保持现有 CSV 输入输出、主入口和配置结构不变，只调整算法内部的数据流、峰值选择逻辑和采样率生效逻辑。

## Key Changes
- FFT 路径
  - 给 `FFTAnalyzer.fft_analyze()` 增加 `peak_search_range` 可选参数；`FreqDynamicAnalyzer` 始终把 `analysis.frequency_range` 传进去，FFT 只在目标基波频带内选峰，不再先全频段取最大峰再做结果过滤。
  - 拆分两份输入数据：`raw_window_data` 作为原始窗口，`fft_input_data` 作为 FFT 峰值检测输入。加窗仅作用于 `fft_input_data`；`FrequencyRefinement.refine()` 一律使用未加窗的 `raw_window_data`。
  - 精化成功时用最小二乘输出覆盖频率、幅值、相位；精化失败时回退到 IpDFT 结果。边界峰值和抛物线分母接近 0 的情况不做插值，直接保留 FFT 峰值。
  - 在 ZC 模式下如果配置里仍开启了 `use_window`、`use_ipdft` 或 `refine_frequency`，只打印一次明确日志说明这些参数当前模式下被忽略，避免误判配置已生效。

- Zero-cross 路径
  - `_detect_zero_crossings()` 改为返回“交点位置 + 边沿方向”，不再只返回一串无类型的 crossing 索引。
  - `rising_only=True` 时只用上升沿到上升沿的整周期计算频率。
  - `rising_only=False` 时分别计算上升沿到上升沿、下降沿到下降沿的整周期，再合并平均；禁止继续用相邻异向 crossing 的半周期直接算频率。
  - `_estimate_phase()` 改为基于“首个 crossing、边沿方向、最终频率、采样率”计算近似相位，并将结果 wrap 到 `[-pi, pi]`。上升沿相位基准为 `0`，下降沿相位基准为 `pi`。
  - 保持 `fft_analyze()` 的返回签名不变；ZC 的相位继续定义为“兼容性近似值”，不承诺与 FFT 相位严格等价。

- 动态分析链路
  - 保留 `window_duration_ms` 和 `step_duration_ms` 作为时间基准，不再把构造时算出的样本点数视为最终真值。
  - 抽出统一的“有效采样率解析”逻辑：当时间戳单调且抖动在可接受范围内时，采用实测采样率作为 `effective_sampling_rate`，并据此重算 `window_samples` 与 `step_samples`；时间戳抖动过大时保留配置采样率并输出风险日志。
  - FFT/ZC 子分析器统一使用 `effective_sampling_rate` 初始化，日志中同时打印“配置采样率 / 实测采样率 / 生效窗口点数”。
  - `load_csv()` 和 `analyze_dynamic()` 都走同一套生效采样率逻辑，确保“从 CSV 读入”和“外部直接传 DataFrame”两种入口行为一致。

## Public Interfaces
- `FFTAnalyzer.fft_analyze()` 新增可选参数 `peak_search_range: Optional[Tuple[float, float]] = None`。
- `analysis.frequency_range` 的语义收紧为“目标基波搜索带 + 输出过滤带”，不再只是结果后过滤条件。
- 不新增 JSON 配置键；现有配置文件继续兼容。

## Test Plan
- FFT 回归
  - 对纯净 `50.0234Hz` 正弦，在 `use_window=True + IpDFT=True + refine_frequency=True` 下覆盖多个初始相位，误差应保持在 mHz 级，不再出现相位相关的大幅漂移。
  - 对 `50Hz` 基波 + 更强 `150Hz` 谐波的合成信号，设置 `peak_search_range=(49.5, 50.5)` 后必须锁定 50Hz，动态分析不能再因为抢到 150Hz 而输出空结果。

- ZC 回归
  - `rising_only=False` 的纯 `50Hz` 正弦必须返回约 50Hz，而不是 100Hz。
  - 对已知相位的纯正弦，`rising_only=True/False` 都应返回有限且一致的近似相位，设置宽松容差用于回归而非高精度计量。

- 动态链路回归
  - 构造“真实 9600Hz、配置 10000Hz”的时间戳数据，动态分析均值应回到接近真实 50Hz，不再系统性偏到约 51.89Hz。
  - 保留真实 CSV 的端到端测试；FFT 精化模式的输出行数应与不精化基线接近，不再因为错误精化大量掉出 `frequency_range`。

- 验证入口
  - 继续保留并运行现有 `python -m pytest tests/test_frequency_refinement.py -q`。
  - 新增独立的 FFT 回归和 ZC 回归 pytest 模块，并纳入同一套测试入口。

## Assumptions
- 本次目标仍是已知窄频带内的工频/近工频检测，不引入通用多音分离或谐波分类器。
- 时间戳可信时，精度优先于强制使用配置采样率；时间戳质量差时回退配置值并明确记录风险。
- 现有 CSV 输出列、`FreqDynamicAnalyzer` 调用方式、以及 FFT/ZC 模式选择优先级保持不变。
